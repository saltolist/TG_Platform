# Метки сводок `1-2-3` и каталог версий

> **Статус:** целевая механика (заменяет `bundleContext`, `rolling_summary_profile.generations`
> и якоря `anchor_user_turn` как источник правды).

Каждое сообщение чата несёт метку **`1-2-3`**, по которой бэкенд однозначно восстанавливает
контекст для LLM. Все тексты сводок хранятся в **каталоге версий** аккаунта (Postgres).

---

## 1. Формат метки

Строка: **`{head}-{attached}-{turn}`**

| Часть | Имя | Смысл |
|-------|-----|--------|
| **1** (`head`) | Головная сводка | Версия bundle в **primer** (`SUMMARY_BUNDLE` в скрытом `user`) на момент этого сообщения |
| **2** (`attached`) | Прикреплённая сводка | Версия bundle, **вшитая в текст user-turn** (floating). `0` — нет прикрепления |
| **3** (`turn`) | Номер user-turn | Позиция в **активной** линии; форк — `3.2`; continuation — `3.2(4)`, `3.2(4.2(5))` |

**Примеры:**

| Метка | Расшифровка |
|-------|-------------|
| `1-0-1` | Первое сообщение, головная сводка v1, без прикрепления |
| `1-2-4` | Голова v1, к 4-му вопросу прикреплена сводка v2 (профиль сменился) |
| `2-0-8` | Голова созрела до v2 |
| `1-3-3.1` | Третий user-turn, **ветка 1** (оригинал), прикреплена v3 |
| `1-0-3.2` | Третий user-turn, **ветка 2** (отредактированный текст), без float |
| `1-0-3.2(4)` | Четвёртый user-turn на ветке `3.2` |
| `1-0-3.2(4.2(5))` | Пятый user-turn после форка на `4.2` внутри ветки `3.2` |

Метка пишется на **user**-сообщение при завершении ответа AI. **ai**-сообщение может
дублировать `turn` и `head` для отладки; сборка промпта ориентируется на user-метки.

Поле в JSON: `contextLabel` (строка). Устаревшее `bundleContext` не используется в новой механике.

## 2. Каталог версий (БД)

Колонка `profiles.summary_catalog` (JSONB), на аккаунт:

```json
{
  "global": [
    {
      "version": 1,
      "text": "## Канал\nТема: Финансы…",
      "fingerprint": "sha256…",
      "createdAt": "2026-06-19T12:00:00.000Z"
    },
    {
      "version": 2,
      "text": "## Канал\nТема: Крипто…",
      "fingerprint": "sha256…",
      "createdAt": "2026-06-19T13:00:00.000Z"
    }
  ],
  "local": {
    "post-uuid-1": [
      {
        "version": 1,
        "text": "## Канал\n…\n## Пост\n…",
        "fingerprint": "sha256…",
        "globalVersion": 2,
        "createdAt": "2026-06-19T14:00:00.000Z"
      }
    ]
  }
}
```

| Тип | Когда создаётся | Что в `text` |
|-----|-----------------|--------------|
| **Глобальная** | `PUT /profile/channel/` (fingerprint канала изменился) | Шаблон: канал + рубрики + аналитика канала |
| **Локальная** | `PATCH /posts/:id/` (fingerprint поста изменился) | Шаблон: канал + **текст поста** + аналитика поста |

Нумерация **1-based**, монотонная внутри `global` и внутри `local[postId]`.
Старые версии **не удаляются** — по метке всегда можно восстановить текст.

**Global chat:** части `head` и `attached` — целые номера из `global[].version`.

**Post chat:** составная метка **`{gHead}.{lHead}-{gAtt}.{lAtt}-{turn}`**:

| Часть | Смысл |
|-------|--------|
| `gHead.lHead` | головная сводка: global (канал) + local (пост) |
| `gAtt.lAtt` | float: global + local на этом turn; `0.0` — нет |
| `turn` | как в global chat |

Примеры post chat:

| Метка | Расшифровка |
|-------|-------------|
| `2.1-0.0-1` | первый turn, канал global v2, пост local v1 |
| `2.3-3.3-4` | head 2.3; после правки канала global созрел до 3, пост v3 |
| `2.3-3.3-3.2(4.2)` | float 3.3 на ветке |
| `11.1-0.2-3` | float только local v2 (пост); global `0` — сводка канала не прикрепляется |
| `11.1-3.0-3` | float только global v3 (канал); local `0` — пост не прикрепляется |

- **Float-слои независимы:** `gAtt=0` — канал не float; `lAtt=0` — пост не float.
- Primer head `gHead.lHead` — полный bundle (local preferred); float — раздельно по слоям.
- **Local** нумерация с **`.1`** — пост уже есть при первом reply.
- **Созревание** global head и local head — **два независимых** счётчика (по N user-turn на ветке).
- Правка **только канала**: `2.3` → после созревания global **`3.3`** (local без изменений).
- Правка **только поста**: `2.3` → `2.4` (global без изменений).
- Старые post-метки `4-7-turn` (без точек) — fallback: local-only, global из `local.globalVersion`.

**Global chat не меняется** — по-прежнему `4-7-3.2(4.2)`.

## 3. Нумерация `turn` и ветки

Активная линия — `flattenVisibleWithPaths(history)` (как в UI).

1. User-turn считается **только по user**-сообщениям активной линии: 1, 2, 3, …
2. Узел **форка** (есть `userBranches[]`): `{n}.{branchIndex + 1}` → `3.1`, `3.2`, …
3. Сообщения **ниже форка** в continuation несут путь предка в скобках:
   `{path}({n})` → `3.2(4)` — четвёртое по счёту, на ветке `3.2`.
4. Вложенный форк в continuation: `{path}({n}.{branch})` → `3.2(4.2)`.
5. Следующие сообщения на этой ветке: `3.2(4.2(5))`, `3.2(4.2(6))`, …

```
U1  → 1-0-1
A1
U2  → 1-0-2
A2
U3 / U3'  (ветки)
  ├─ 3.1: «текст A»  → 1-0-3.1  + хвост 3.1(4)…
  └─ 3.2: «текст B»  → 1-0-3.2  + хвост 3.2(4)…
U4  → 1-0-3.2(4)     (на активной ветке 3.2)
U4' → 1-0-3.2(4.2)   (редактирование U4, ветка 2)
U5  → 1-0-3.2(4.2(5))
```

При переключении `activeUserBranch` меняется видимая метка turn на узле форка;
**каталог сводок общий** на аккаунт, состояние головы — **per thread** (см. §5).

## 4. Динамические сводки через метки

Механика «догоняния» сохраняется, но выражается **явно**:

1. Правка профиля → новая глобальная версия `G_new` в каталоге.
2. На ближайшем user-turn метка получает `attached = G_new`, `head` = ещё старая.
3. После **N** user-turn (по умолчанию 5) на **этой ветке** `head` созревает: `head = G_new`, `attached = 0`.
4. В primer попадает текст `catalog[head]`; float — `catalog[attached]` на соответствующем turn в окне.

Никаких `anchor_user_turn` / `stub_generation_id` в промпте — только числа в метках и каталог.

**Двойная правка профиля:** в каталоге `G2`, `G3`; на turn сначала `1-2-4`, потом `1-3-4`,
созревание по очереди: сначала `head=2`, затем `head=3`.

## 5. Ветки и `thread_context`

В метаданных чата (`thread_context[thread_key]`):

```json
{
  "head_version": 1,
  "pending_version": 2,
  "pending_since_turn": 4,
  "rolling_summary": "",
  "rolling_summary_idx": 0
}
```

| Событие | Действие |
|---------|----------|
| Форк на turn `k` | Новый `thread_key`; `head`/`pending` **унаследованы с обрезкой** по turn ≤ k; саммари диалога — пустое |
| Ответ AI на ветке | Метка на user-msg; обновление `thread_context` этой ветки |
| Переключение ветки | Сборка из `thread_context[active_key]` + метки сообщений **активной** линии |
| Rolling summary | Как раньше, **только** по активной линии; в primer отдельным блоком `CONTEXT_SUMMARY` |

**Правило форка:** метки на сообщениях **ветки 0** не меняются; ветка 1 получает новые метки
при новых ответах AI. Сводки берутся из каталога по числам в метке — не из родительского timeline.

## 6. Сборка окна LLM

```
1. system  ← systemPrompt
2. primer  ← catalog.resolve(scope, head=H) + CONTEXT_SUMMARY (если есть)
3. assistant «Понял, учту при ответах.»
4. окно    ← последние K пар; у user с attached≠0 дописать SUMMARY_BUNDLE
5. last user ← текущий запрос (+ RAG)
```

Для сообщения в окне с меткой `1-2-5`:

```text
[user] SUMMARY_BUNDLE:
       {текст версии 2 из каталога}

       {текст вопроса turn 5}
```

Primer для этого же запроса использует **версию 1** (head текущего thread state или последнего user в контексте).

## 7. API и запись меток

| Событие | Каталог | Метка |
|---------|---------|-------|
| `PUT profile/channel` | +global version при смене fingerprint | — |
| `PATCH posts/:id` | +local version при смене fingerprint поста | — |
| `POST ai/reply` (конец стрима) | — | stamp `contextLabel` на последний user-msg; обновить `thread_context` |

Клиент **не** вычисляет метки — только отображает при отладке (опционально).

---

## 8. Миграция с текущей схемы

1. Чаты без `contextLabel` → fallback на старую сборку (временно).
2. При первом `PUT channel` после деплоя — `global[1]` из текущего профиля.
3. Старые `bundleContext` / `rolling_summary_profile` игнорируются, если у сообщения есть `contextLabel`.

---

## 9. Инварианты

1. `head` и `attached` всегда указывают на **существующие** версии в каталоге (или `attached=0`).
2. На одном user-turn **не больше одного** прикреплённого bundle (`attached` одно число).
3. Ветка не использует `head` выше версии, которая могла созреть на её длине (обрезка при форке).
4. Глобальный и локальный каталоги не смешиваются в одной метке post/global chat.
