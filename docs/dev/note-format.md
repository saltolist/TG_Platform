# Формат заметки

> Статус: **зафиксировано**. Реализация — [Фаза 2](../backend/phases/phase-2-ai.md) (часть A).

Заметки в TG Platform хранятся в виде объектов с метаданными и markdown-телом.
Этот документ является каноном для миграции, рендера и RAG-индексации.

---

## Объект заметки

```json
{
  "id": "string",
  "title": "string",
  "date": "ISO-8601 string",
  "ai": false,
  "body": "CommonMark + GFM markdown",
  "files": [
    { "id": "string", "name": "string", "type": "string", "url": "string" }
  ]
}
```

- **`title`** — заголовок заметки; отображается отдельно, **не** включается в `body` как `# Заголовок`.
- **`date`** — дата создания/изменения, ISO-8601.
- **`ai`** — флаг, создана ли заметка ассистентом.
- **`body`** — текстовое тело в формате [CommonMark](https://spec.commonmark.org/) с расширением [GFM](https://github.github.com/gfm/) (таблицы).
- **`files`** — реестр прикреплённых медиафайлов; идентифицируются по стабильному `id`.

Метаданные хранятся как поля объекта, **не** как YAML frontmatter внутри `body`.

## Синтаксис тела (`body`)

### Поддерживаемые конструкции

| Конструкция | Синтаксис | Примечание |
|---|---|---|
| Жирный | `**текст**` | |
| Курсив | `*текст*` | |
| Заголовки | `# H1` … `###### H6` | |
| Списки | `- пункт` / `1. пункт` | |
| Цитата | `> текст` | |
| Код (строчный) | `` `код` `` | |
| Код (блок) | ` ```язык … ``` ` | |
| Горизонтальная линия | `---` | |
| Ссылка | `[текст](url)` | |
| **Таблица (GFM)** | `\| A \| B \|` / `\|---|---\|` … | Полная поддержка GFM-таблиц |
| **Картинка-вложение** | `![alt](attachment:<id>)` | `<id>` — из реестра `files` |
| **Файл-вложение** | `[имя](attachment:<id>)` | `<id>` — из реестра `files` |

### Медиа-вложения (attachment)

Ссылка `attachment:<id>` указывает на запись в `files` с соответствующим `id`.
Ссылка **не содержит** отображаемое имя или URL — они берутся из реестра `files` по `id`.
Это делает ссылку устойчивой к переименованию файла.

**Картинка:**
```markdown
![Скриншот портфеля](attachment:f3a9b1c2)
```

Рендерер ищет в `files` запись `{ id: "f3a9b1c2", url: "...", name: "..." }` и показывает изображение (или чип с именем, если `url` недоступен).

**Файл (не изображение):**
```markdown
[Отчёт за апрель](attachment:7e22d001)
```

Рендерер показывает чип с `name` из реестра и ссылкой на `url`.

### Сетка картинок

Несколько картинок **подряд** (без пустой строки между ними) рендерер группирует в горизонтальный ряд (до 3 в ряду). Пустая строка или любой другой элемент разрывают группу.

```markdown
![Фото 1](attachment:aaa)
![Фото 2](attachment:bbb)
![Фото 3](attachment:ccc)

Следующий абзац. Картинки выше отображаются в ряд.
```

Правило применяется **только рендерером** — в `body` всегда хранится плоский markdown.
Максимальное число картинок в ряду: **3** (константа `MAX_IMAGES_PER_ROW = 3`).

## Таблицы (GFM)

Полный синтаксис GFM-таблиц:

```markdown
| Актив    | Доля  | Доходность |
|----------|-------|------------|
| Акции    | 60%   | +12%       |
| Облигации| 30%   | +6%        |
| Кэш      | 10%   | +3%        |
```

Выравнивание через `:`:

```markdown
| Левый | Центр | Правый |
|:------|:-----:|-------:|
| текст | текст |  текст |
```

## Реестр файлов (`files`)

Каждый элемент:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | string | Стабильный UUID/slug; используется в `attachment:<id>` |
| `name` | string | Отображаемое имя (чип, alt-текст по умолчанию) |
| `type` | string | MIME-тип (`image/jpeg`, `application/pdf`, …) |
| `url` | string | Публичный URL (MinIO presigned или CDN) |

Поле `id` **обязательно** для любого файла, на который ссылается `body`.
Файл без `id` в теле не упоминается и хранится только как вложение без ссылки.

## Текстовый индекс для RAG

При индексации для RAG из `body` извлекается чистый текст:
- markdown-разметка убирается;
- alt-текст картинок (`![alt](...)`) **сохраняется**;
- `attachment:<id>` вырезается;
- текст ячеек таблиц **сохраняется**;
- `title` добавляется в начало индексируемого текста.

Формат для индексатора: `"<title>\n\n<plain text из body>"`.

---

## Устаревший формат (до миграции)

До перехода на markdown `body` хранил строку с кастомными embed-токенами вида `[имя]`,
где `[имя]` ссылался на файл **по имени** (не по `id`) через реестр `files[].name`.

После миграции A2:
- все `[имя]` конвертированы в `![имя](attachment:<id>)` или `[имя](attachment:<id>)`;
- каждому файлу гарантирован стабильный `id`;
- текстовые части экранированы от конфликтов с markdown-синтаксисом.
